# Ambient Dream 双界面版本（UI Profile）切换设计稿

## 背景与目标

在保留现有界面体验（Classic）的前提下，引入一套“改进呈现方式”的界面版本（Refined）。用户可通过界面按钮随时切换两版；默认策略为“跟随上次选择”。两版共享同一套数据与功能，只改变呈现与交互密度，便于用户按当下心境选择。

## 范围

### 必做

- 新增 `uiProfile: 'classic' | 'refined'` 并持久化到 localStorage
- 默认策略：若存在历史选择则沿用；否则默认 `classic`
- 在“右上收藏面板旁”提供一个可见的切换按钮
- Classic：尽可能保持现状，不做行为改变
- Refined：在不改数据结构的前提下，提供更沉浸、更低工具感、更可达的呈现方式（渐进披露、减少同屏常驻面板存在感、文案口径收敛等）

### 不做（本阶段）

- 不拆分成多个 HTML 文件
- 不引入账号/云同步等体系
- 不在本阶段重写信息架构，只做“呈现与交互密度”的改版

## 核心设计原则

- 用户可控：切换是即时的、可逆的、不会丢数据
- 同一内容，两种气质：Classic 保留“功能全开/展示性”的可玩感；Refined 更偏“睡前放松”的低刺激
- 不让切换变成“设置管理”：入口轻量、文案温柔、解释一句就够

## 状态模型与存储

### State

- `uiProfile`: `'classic' | 'refined'`

### Storage Key

- `ambient-dream-ui-profile`

### 初始化逻辑

- 读取 `localStorage['ambient-dream-ui-profile']`
  - 若值为 `'classic' | 'refined'` 则使用该值
  - 否则默认 `'classic'`

### 持久化逻辑

- 任意切换后立即写回 localStorage

## 入口与交互

### 入口位置

- 右上收藏面板旁（与 Favorites 面板同一区域）

### 入口形态

- 小胶囊按钮（低存在感、可点击清晰）
- 文案建议（两套之一即可）：
  - `经典 / 沉浸+`
  - `原版 / 改进版`
- Tooltip/辅助说明（hover 或长按时出现）：
  - “切换界面呈现，不影响内容”

### 行为

- 点击切换 `uiProfile`
- 切换后短暂 toast（可选，2s 内淡出）：
  - “已切换到 经典” / “已切换到 沉浸+”

## 技术呈现策略（单文件 UI Profile）

### Class 注入

- 在 `.main-stage` 根节点增加 profile class：
  - `ui-classic` / `ui-refined`

### 组织方式

- CSS：用 `.ui-refined ...` 覆盖 Classic 样式，做到“Classic 不动、Refined 叠加”
- React 渲染：尽量用现有结构，通过 `uiProfile` 控制：
  - 某些模块是否渲染
  - 某些控件在阅读模式是否折叠
  - 文案字符串选择

## 两版差异定义

### Classic（现状版）

- 控件、面板、文案尽量保持现状
- 不引入新的折叠/抽屉逻辑（避免让“原版用户”感到变化）

### Refined（改进版）

#### 阅读模式下（优先）

- 降低同屏“工具存在感”
  - 右下 controls 更“轻”：默认展示更少入口或更低 opacity
  - 音乐坞/收藏面板默认更隐身，触发时才显著展开
- 渐进披露：将“强编辑工具”从阅读态移走或降级为二级入口

#### 编辑/整理模式下（其次）

- 对“整理态工具密度”做收拢：
  - 分组/筛选/搜索收进一个“整理面板”入口（点击展开/收起）
  - 默认只保留：退出整理、上一层/下一层、当前层信息

#### 文案口径

- Refined 下收敛中英混用（倾向统一中文或极克制双语）
- 目录/整理等工具词在阅读态更温柔（例如以“线索/拾起/再相遇”等短词替代强管理语气）

#### 无障碍与舒适性（建议两版都收益）

- 补齐 `:focus-visible` 焦点可见样式
- 色弱友好：选中态不只靠颜色点，提供形状/符号/粗体等冗余线索
- 触控命中区不低于 40–44px（尤其是小按钮）

## 验收标准

- 默认跟随上次选择；清空 localStorage 后默认 Classic
- 切换不刷新页面、不丢失便签/语录/音乐播放列表等状态
- 切换按钮在桌面与移动端都可触达、可点击、文案清楚
- Classic 切换前后视觉与交互保持一致（允许仅增加切换按钮本身）
- Refined 相比 Classic 明显更“隐身/沉浸”，工具密度降低，且可达性更好
