## 目标

在不破坏“温柔、梦幻、诗意”的基调下，增强五类体验：

1. 自读的时间感：正文下方加入极轻的“呼吸式进度”（细线 + 游走点）。
2. 触感：关键按钮加入“果冻般的柔软按压反馈”，且按控件层级分层处理。
3. 再次相遇：提供一个轻量入口，让收藏/标签整理的结果能被温柔再利用。
4. 旁白式提示：意象文案先出现，再缓缓显出真实操作提示（必要时直接显示真实提示）。
5. 四季沉浸背景：根据当前便签数量切换春夏秋冬，提供强氛围的动态场景（多层视差 + 季节粒子 + 可交互涟漪），并内置性能/无障碍降级策略。

## 非目标

- 不引入大型背景图片资源依赖（避免加载延迟与带宽抖动）。
- 不引入会触发布局重排的动画（禁止布局属性动画，保持 GPU-friendly）。
- 不改变现有功能逻辑与数据结构（收藏、目录、便签等保持可用）。

## 设计原则

- 沉浸优先，工具隐身：默认状态更像“环境”，功能在需要时才显形。
- 动效克制但“有生命”：不跳、不躁，不用高饱和霓虹，不用夸张发光。
- 可用性优先于诗性：当用户开始操作时，真实提示优先出现，避免误导。
- 只用 `transform` / `opacity` 驱动视觉动效；尊重 `prefers-reduced-motion`。

## 规格概览

### A. 旁白式提示：意象 → 真实提示

#### 触发点

- 收藏空态、目录提示、自读提示、随机入口提示等。

#### 文案结构

- 每条提示定义两段文案：
  - `imagery`：意象旁白（先显示）
  - `literal`：真实操作提示（后显示）

#### 显示行为

- 默认：先显示 `imagery`，停留 1.6–2.2s 后开始交叉渐变到 `literal`（总过渡 1.2–1.6s）。
- 操作优先：在以下情况下直接显示 `literal`：
  - 鼠标移入相关面板/区域
  - 面板展开
  - 按钮获得键盘焦点（focus-visible）

#### 文案示例（可替换）

- 收藏空态：
  - imagery：喜欢的句子，会在这里慢慢聚拢。
  - literal：把你喜欢的句子收藏在这里。
- 自读提示（自读开启时正文下方微提示）：
  - imagery：让它自己读下去，也让你慢一点。
  - literal：自读已开启，再次点击可关闭。
- 目录提示：
  - imagery：你写下的，都在这里等你翻回。
  - literal：打开目录，查看与整理语录。

### B. 自读呼吸式进度（正文下方）

#### 目标

让用户“感觉到时间在流动”，但不会被“计时器/进度条”的工具感打断。

#### 位置

正文下方、现有提示文本附近（同一视觉带，层级更低）。

#### 形态

- 细横线：宽度约 120–160px，高度 1px（或视觉等效），轻微呼吸。
- 游走点：一个小点沿线左右缓慢游走，不触碰边缘（留 8–12px 的安全边界）。

#### 显示规则

- 仅在自读开启时显示。
- 当用户进行明显操作（滚轮/拖拽/点击正文区域）时淡出；静置 1.2s 再淡入。

#### 节奏绑定

- 呼吸周期与自读切换节奏一致或近似（建议按当前自读间隔计算一个“呼吸周期”），避免割裂。

### C. 果冻触感（按层级分层）

#### 分层规则

- 大按钮（高频/关键入口）：沉浸+/经典、播放、目录、自读、♥ 等
  - 采用“轻果冻”：按下下沉 1px + scale 0.98 + 透明度轻降 + 雾面高光微调；释放平滑回弹（不 overshoot）。
- 小按钮（工具性/密集）：📌、mini 按钮、列表内小按钮等
  - 采用“只软不弹”：仅下沉 1px + 透明度变化，不做缩放。

#### 交互一致性

- `:active` 与 pointer 事件的一致性优先：触控设备必须有同等反馈。
- `prefers-reduced-motion`：保留按下态（瞬时反馈），取消缓慢的回弹动效。

### D. “再次相遇”入口（正文下方）

#### 目标

让收藏/标签不只是“管理”，而是“再次相遇”的路径。

#### 形态

正文下方轻量文字按钮（像一句邀请），示例：再遇见一条。

#### 行为

- 如果存在收藏：随机选取一条收藏并切换到它（且给一个极轻的旁白式确认提示）。
- 如果没有收藏：按钮不打断（不弹硬提示），触发旁白式提示（意象 → 真实提示）引导用户收藏。

#### 扩展（可选，暂不实现）

长按/右键打开极简菜单：从收藏 / 从标签 / 从全部。默认版本先只做“从收藏随机”。

### E. 四季沉浸背景（春夏秋冬）

#### 触发规则（按“当前便签数量”）

- 计数口径：当前漂浮便签数量（现存 notes 数量）。删除会减少计数，可能触发季节回退。
- 阶段：
  - 0–5 张：保持现有背景不变
  - 6–13 张：春
  - 14–21 张：夏
  - 22–29 张：秋
  - 30–37 张：冬
  - 之后每 +8 循环

#### 防抖与过渡

- 季节切换 debounce 300ms（计数稳定后再切换）。
- 切换过渡 1–2s：色彩、雾层、粒子密度与交互涟漪风格交叉渐变，不硬切。

#### 场景基调（色彩与光影）

- 春：嫩绿 + 樱花粉，柔和晨光光晕。
- 夏：深蓝 + 紫罗兰，萤火虫般闪烁光点。
- 秋：金黄 + 暖橙，飘落枯叶与夕阳余晖。
- 冬：冰蓝 + 银灰，飘雪与冷冽雾气。

#### 多层视差结构

- 远景：大范围雾化渐变与光晕（低频缓慢呼吸）。
- 中景：模糊季节性形体（云、枝影、植物轮廓）缓慢漂移。
- 近景：季节粒子系统（前景飘落/漂浮，层次最强）。
- 视差：鼠标移动驱动三层不同速率偏移；移动端弱化或关闭。

#### 动态粒子系统（VFX）

- 春：花瓣（更轻、更慢，带轻微旋转）。
- 夏：流萤（点状光粒，随机闪烁，偶尔聚散）。
- 秋：落叶（重一点、速度更不均，偶尔小旋涡）。
- 冬：飞雪（细小、密度可调，带冷雾层）。

#### 交互涟漪（点击/拖拽）

空白处点击生成季节涟漪：

- 春：花瓣扩散（由中心向外散开并消隐）。
- 夏：光点弹跳（几次弹跳后回到漂浮）。
- 秋：落叶旋涡（短暂环流后解散）。
- 冬：冰霜蔓延（以“扩散边缘”的方式表达，随后淡去）。

拖拽空白区域：产生短暂“扰动带”，粒子产生轻微环流/避让。

#### UI 微调（随季节）

- 便签边缘/阴影随季节轻微调整：
  - 冬：边缘更冷、更清（类似薄霜感，但不发光）
  - 春：阴影更柔、更暖（轻微花纹投影感）
- 播放控制区的 hover/active 色调随季节微调，保持一致性（不改变布局）。

## 性能与无障碍（必须实现）

- 动效只用 `transform` / `opacity`（禁止布局属性动画）。
- 支持 `prefers-reduced-motion`：
  - 冻结粒子、视差、涟漪
  - 保留静态季节氛围色与基础过渡（可缩短/减弱）
- 内置降级策略：
  - FPS < 30 或设备压力大：降低粒子数量、关闭涟漪交互，保留季节色块与雾层。
- 事件节流：
  - 鼠标/触控驱动的视差采用 `requestAnimationFrame` 合帧
  - 季节切换 debounce 300ms

## 验收标准

- 自读开启：正文下方出现细线 + 游走点，且在用户操作时自动淡出、静置后淡入。
- 果冻触感：大按钮按下有轻果冻，小按钮按下只有软按压；整体无跳动、无弹跳。
- 再次相遇：正文下方可触达入口；有收藏时随机切换；无收藏时仅旁白式引导。
- 旁白式提示：默认先意象后真实；在用户开始操作时真实提示优先显示。
- 四季背景：0–5 保持现有；6 开始春；之后每 +8 轮换；切换始终 1–2s 平滑过渡。
- `prefers-reduced-motion` 开启时所有动态效果冻结，页面仍美观可用。
