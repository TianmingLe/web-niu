## 背景

手机端屏幕高度有限，当前页面的音乐坞（music-dock）、底部按钮组（controls）与中部“再遇见一条/滚动提示”（bottom-center-stack）会在部分机型上发生遮挡与堆叠，影响观感与可用性。

同时，手机端第一次进入时用户不容易理解页面的全部能力（翻句、收藏/目录、音乐播放等），需要一次“功能亮相”的引导，但不能长期占用屏幕空间。

## 目标

- 手机端不发生组件遮挡：音乐坞、底部按钮组与中部提示互不重叠
- 音乐坞展开时页面清爽：展开仅保留音乐相关内容，其它控件自动隐藏；收起后恢复
- 首次进入提供“逐步亮相 → 收纳”的功能引导，并且引导期间可点击，用户交互后立刻进入正常布局
- 电脑端视觉与布局完全不改动

## 非目标

- 不重做视觉风格、字体、颜色体系
- 不增加复杂新功能（例如菜单页、二级路由）
- 不改变现有桌面端交互（滚轮/上滑切换、自读、收藏、目录、整理、音乐播放逻辑）

## 设备检测与分流策略

以 viewport 为准，不依赖 UA：

- `isMobile = window.innerWidth <= 820`
- `isPortrait = window.innerHeight > window.innerWidth`

将结果映射到 `.main-stage` 的 class：

- `viewport-mobile`（手机）
- `viewport-mobile-portrait`（手机竖屏）

更新时机：

- 首次渲染后立即计算一次
- 监听 `resize` 与 `orientationchange` 重新计算（节流或合并到已有的 viewportProfile 更新机制中）

## 手机端最终布局（两排收纳）

约束：该布局仅在 `viewport-mobile` 下启用。

### 底部第 1 排（最底）

目标：让“主要操作”稳定可达，且不和音乐坞抢空间。

常驻按钮（用户选择 B）：

- 收藏（♥）
- 目录
- 播放（与音乐坞的播放状态一致）

表现：

- `.controls` 在手机端改为紧凑栅格/两排布局（或单排等宽按钮），固定在底部安全区之上
- `自读/整理` 在手机端不常驻，可通过目录或其它入口触达（不在本设计稿范围内新增入口）

### 底部第 2 排（上方一排）

目标：在底部第 1 排之上，容纳音乐坞收起态 + 中部提示，不遮挡正文。

内容：

- `.music-dock` 默认折叠态保持“紧凑宽度”（避免占满整条屏幕）
- `.bottom-center-stack`（提示 + 再遇见一条）保持水平居中

### 音乐坞展开时的隐藏规则

触发条件：手机端且 `musicDockOpen || musicDockPinned`（即 `.music-dock.expanded` 生效）

规则（用户确认）：

- 自动隐藏 `.controls`（底部按钮组）
- 自动隐藏 `.bottom-center-stack`（提示 + 再遇见一条）
- 音乐坞收起后恢复显示

## 手机端首次引导动效（逐步亮相 → 收纳）

### 触发条件

- 仅 `viewport-mobile` 生效
- 默认仅首次进入触发（用 `localStorage` 记录一次性标记）
- reduced-motion 用户直接跳过（或仅保留一次极轻提示）

建议键名：

- `ambient-dream-mobile-onboarding-v1`

### 交互原则（允许点击）

- 引导期间不设置遮罩层，不阻断 pointer events
- 任意用户交互即中止引导并进入最终布局：
  - `pointerdown` / `wheel` / `touchmove` / `keydown`
  - 或者任何与音乐/目录/收藏相关的点击
- 中止后写入一次性标记，避免重复触发

### 动效脚本（建议节奏）

总时长控制在 2–3 秒内，避免“演示过长”影响正文阅读。

依次点亮三块能力（尽量用 `opacity / scale / glow`，避免大位移）：

1. 音乐：突出音乐坞折叠态播放按钮（提示“点一下播放”）
2. 翻句：短暂点亮“上滑/下滑提示 + 再遇见一条”
3. 功能区：短暂展开底部按钮组为更可见形态，然后收纳为最终紧凑态

完成后落到“手机端最终布局（两排收纳）”。

## 视觉与可用性约束

- 不遮挡正文：引导动效仅影响控件本身的透明度/发光/缩放，不在正文区域增加额外层
- 动效强度上限：避免强闪烁；发光/描边透明度克制
- 兼容浏览器 autoplay 限制：若自动播放失败，toast 提示优先级高于引导动效，且引导应在用户点击后立刻结束

## 验收标准

- 手机端竖屏：收起态不发生遮挡（音乐坞、底部按钮组、中部提示互不重叠）
- 手机端音乐坞展开：`.controls` 与 `.bottom-center-stack` 自动隐藏；收起后恢复
- 引导动效：
  - 首次进入触发且只触发一次
  - 动画期间可点击，点击任一控件立即转入正常布局且操作生效（例如点击播放立即播放）
  - reduced-motion 下不播放或显著减弱
- 电脑端：视觉布局无变化（回归对比截图/手动验收）
